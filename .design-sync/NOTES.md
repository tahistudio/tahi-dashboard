# design-sync notes — tahi-dashboard

Repo-specific gotchas for syncing the Tahi design system to claude.ai/design.
Read this first on any re-sync.

## What this repo is

`tahi-dashboard` is a **Next.js app**, not a published component library. There is
no component `dist/` and no barrel export. The converter therefore runs in
**`package` shape with an explicit barrel `--entry`** (`cfg.entry =
.design-sync/ds-entry.tsx`) that re-exports ONLY the curated design-system
primitives. This keeps the esbuild IIFE decoupled from the app's
server/Next/Clerk code.

- Curated set = the components showcased in
  `app/(dashboard)/design-system/design-system-content.tsx` (its import block,
  lines 5-40, is the authoritative export-name list) + `ToastProvider`.
- 46 components across 24 source files in `components/tahi/`, grouped:
  brand(5), icons(9), primitives(14), charts(10), data(2), messaging(4), boards(2).
- `window.TahiDS` is the runtime global.

## Regenerating inputs

All build inputs are generated by two scripts (commit their OUTPUT, re-run when
the source changes):

1. `node .design-sync/setup.mjs` — writes `.design-sync/ds-entry.tsx` (barrel),
   `.design-sync/stubs/*` (Next stubs), `tsconfig.dssync.json`,
   `.design-sync/docs/<Name>.md` (group frontmatter + extracted header JSDoc),
   and `design-sync.config.json`. Edit the `M` manifest in setup.mjs to add or
   regroup components, then re-run.
2. `node .design-sync/compile-css.mjs` — REQUIRED before every build. Compiles
   `app/globals.css` (Tailwind v4) into `.design-sync/compiled.css`. The brand
   tokens live in a Tailwind `@theme {}` block that browsers ignore until
   Tailwind turns it into real `:root` custom properties, so shipping raw
   globals.css would render every component token-less. Uses the repo's own
   `@tailwindcss/postcss`. `compiled.css` is gitignored — always regenerate.

## Decisions / gotchas

- **Next.js imports** (`next/link` in Card, FeatureCard, Menu; plus `next/image`,
  `next/navigation` as insurance) are resolved to inert stubs via
  `tsconfig.dssync.json` paths + the converter's tsconfig-paths plugin.
  `next/link` -> a plain `<a>`. No curated component uses Clerk or `@/lib/db`.
- **Transitive internal deps**: the curated set pulls in `input.tsx` and
  `popover.tsx` (used by board-view/data-table/filter-bar/menu/message-bubble).
  Both are pure presentational and bundle fine; they are internal (not exported
  on `window.TahiDS`, not separate cards).
- **Heavy deps that bundle**: `motion/react` (animated icons), `recharts`
  (charts), `@tiptap/*` (Composer, MessageBubble). Watch the `_ds_bundle.js`
  size against the 5 MB upload cap.
- **Tokens vs Tailwind**: primitives (Badge, Card, etc.) are inline-styled with
  `var(--token)`; the heavier components (DataTable 13, FilterBar 11, BoardView 8,
  KanbanBoard 5) also use Tailwind utility classes — both are covered by
  `compiled.css`.
- **Font**: Manrope loads at runtime via the Google Fonts `@import` at the top of
  `compiled.css`. `cfg.runtimeFontPrefixes: ['Manrope']` suppresses FONT_MISSING.
  The repo ships no local Manrope file.
- **Weak `.d.ts`**: no shipped types, so prop contracts are synthesized and thin.
  Hand-write `cfg.dtsPropsFor.<Name>` for components whose API matters most
  (compounds, key primitives) using their TS source interfaces.
- **Toast**: exported as `ToastProvider` + `useToast` (no standalone component).
  Its visible toast only renders after `showToast()` — its preview triggers on
  mount with a long duration.

## Re-sync risks (what can silently go stale)

- `compiled.css` is regenerated, not committed — a re-sync that skips
  `compile-css.mjs` ships stale tokens/utilities. Always re-run it.
- The barrel/docs/config are generated from the `M` manifest in setup.mjs. If a
  component is renamed/removed in `components/tahi/`, update `M` and regenerate,
  or the build fails resolving the missing export.
- Export names are pinned by hand in `M` (no `.d.ts` to validate against). If an
  upstream component changes its exported name, the barrel breaks loudly at build.
- `next/*` stubs are minimal; if a newly-added curated component uses a Next API
  the stubs don't cover, extend the stubs + tsconfig.dssync.json paths.
- Authored previews under `.design-sync/previews/` are tied to the components'
  current props — an upstream prop rename can make a preview render wrong.

## Preview authoring notes (wave 1 — all 46 graded good)

Folded from the per-batch learnings files. Re-read before re-authoring.

- **Portal/overlay components.** Four components render outside a normal card:
  - `Tooltip`, `Menu` — open only on hover/click and portal to `document.body`,
    with NO `open`/`defaultOpen` prop. They cannot render their open state in a
    static screenshot. Their preview cards show a faithful **pinned-open
    representation** (real trigger + a styled bubble/panel using the component's
    own tokens). The bundled `Tooltip`/`Menu` the design agent imports are the
    real components; the card is illustrative. Documented, intentional.
  - `SlideOver` (has `open` prop) and `ToastProvider` (toast via `showToast`)
    DO render for real — `cfg.overrides` gives them `cardMode:"single"` so the
    card becomes the containing block for their `position:fixed` panels. Primary
    cells: SlideOver→`LiveOpenPanel`, ToastProvider→`ToastSuccess`.
- **Avatar** wraps itself in `<Tooltip>` by default (`tooltip` defaults true);
  pass `tooltip={false}` in compositions where the Avatar isn't the subject.
- **lucide-react is NOT on `window.TahiDS`** — preview cells that need an icon
  use inline `<svg>` (the `Animated*` icon components ARE exported and used as
  the icon set; brand marks via `LeafIcon`/`LeafGlyph`).
- **Recharts charts** (`BarChart`, `LineChart`, `MultiBarChart`) need an explicit
  pixel/`rem` wrapper width — `ResponsiveContainer` measures the parent. The
  custom-SVG charts (`DonutChart`, `Gauge`, `Sparkline`, `FunnelChart`,
  `GanttChart`, `Heatmap`, `CalendarHeatmap`) tolerate fluid wrappers.
- **Tiptap** (`Composer`, `MessageBubble`) renders statically fine: MessageBubble
  uses `dangerouslySetInnerHTML(bodyHtml)` at rest (pass HTML like `<p>…</p>`);
  Composer's editor paints its placeholder. No override needed.
- **IntersectionObserver** animations (`ProgressBar`, charts) fire in the
  Playwright capture, so bars/arcs reach their target values.
- Detailed per-component prop shapes (DataTable columns/`getRowId`, KanbanBoard
  `BoardItem`/`status`-keying, FilterBar `FilterDef`/`ActiveFilter`, GanttChart
  `GanttRow`, MessageBubble/Composer/FileAttachment shapes) were captured during
  authoring — read the authored `.design-sync/previews/<Name>.tsx` as the
  worked reference; they import only from bare `'tahi-dashboard'`.
